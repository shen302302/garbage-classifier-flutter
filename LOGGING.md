# 日志功能说明

## 功能概述

本应用已集成完善的日志系统，支持多级别日志记录、文件存储、日志导出等功能。

## 日志级别

- **DEBUG**: 详细的调试信息，用于开发阶段
- **INFO**: 一般信息，记录应用运行状态
- **WARNING**: 警告信息，可能的问题但不影响运行
- **ERROR**: 错误信息，需要关注的问题
- **OFF**: 关闭日志记录

## 日志存储

- 日志文件存储在应用文档目录的 `logs` 文件夹中
- 每天一个日志文件，格式：`app_YYYY-MM-DD.log`
- 当日志文件超过 5MB 时会自动轮转
- 轮转的日志文件会移动到 `archive` 目录

## 设置配置

在设置页面可以配置：

1. **日志开关**
   - 启用或禁用日志记录

2. **日志级别**
   - 选择记录哪些级别的日志
   - INFO 级别包含所有 WARNING 和 ERROR 级别的日志

3. **保留天数**
   - 设置自动清理日志的保留天数
   - 范围：1-30天

## 日志导出

1. 点击"导出日志"按钮
2. 系统会打包所有日志文件为 ZIP 格式
3. 可以通过分享功能发送给开发者

## 日志清理

1. 自动清理：应用会在每天凌晨2点自动清理超过保留天数的日志
2. 手动清理：在设置页面点击"清除日志"按钮

## 使用示例

```dart
import 'package:ultralytics_yolo/utils/logger_manager.dart';

// 记录不同级别的日志
logger.d('Debug 信息');
logger.i('应用启动');
logger.w('警告信息');
logger.e('错误信息', error: Exception('错误详情'), stackTrace: StackTrace.current);
```

## 性能影响

- 日志记录对应用性能影响很小
- 建议在生产环境中使用 INFO 或 WARNING 级别
- ERROR 级别的日志会记录堆栈信息，可能占用较多存储空间

## 安全注意事项

- 日志文件可能包含敏感信息
- 导出日志前请确保内容可以分享
- 定期清理不必要的日志文件
